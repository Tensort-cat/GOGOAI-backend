# Yuko Chat 系统设计文档

## 1. 项目简介
Yuko Chat 是一个基于 Golang 实现的即时通讯系统，采用前后端分离架构。系统支持单聊、群聊、消息持久化以及 WebSocket 实时通信。

## 2. 技术栈
- 后端：Golang + Gin + GORM
- 数据库：MySQL
- 缓存：Redis
- 消息队列：Kafka
- 前端：Vue3 + Vuex
- 部署：Docker + Nginx

## 3. 系统架构

### 3.1 网关层
负责处理客户端请求，包括 HTTP 和 WebSocket 请求，并进行统一鉴权。

### 3.2 业务层
主要负责消息处理逻辑，包括：
- 消息发送
- 消息存储
- 消息同步

### 3.3 数据层
负责数据持久化：
- MySQL：存储用户信息和聊天记录
- Redis：缓存在线状态和会话信息

## 4. 消息流程

1. 客户端通过 WebSocket 发送消息
2. 网关层接收并校验 token
3. 消息发送到 Kafka
4. 消费者从 Kafka 读取消息并写入 MySQL
5. 同时将消息推送给接收方

## 5. 用户系统

### 5.1 用户注册
用户通过手机号注册，密码使用 bcrypt 加密存储。

### 5.2 用户登录
登录成功后返回 JWT Token，用于后续请求鉴权。

## 6. 会话管理
系统支持单聊和群聊两种会话：
- 单聊：两个用户之间的私密对话
- 群聊：多个用户组成的聊天组

## 7. 扩展功能

### 7.1 文件存储
系统使用 MinIO 进行文件存储，支持图片和文件上传。

### 7.2 消息搜索
支持基于关键词的历史消息搜索功能。

## 8. 性能优化

- 使用 Redis 缓存热点数据
- Kafka 削峰填谷
- 数据库分表（按会话 ID 分表）

## 9. 安全设计

- 使用 JWT 做身份认证
- 使用 HTTPS 保证传输安全
- 敏感数据加密存储

## 10. 部署方式

系统使用 Docker 部署，主要服务包括：
- MySQL
- Redis
- Kafka
- 后端服务
- 前端服务

通过 Nginx 做反向代理和负载均衡。
